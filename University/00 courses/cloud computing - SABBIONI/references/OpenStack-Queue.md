Usually rabbitMq but can be switched to [Apache Qpid](https://qpid.apache.org/) for decoupling in space (sender doesn't know receiver and viceversa) and time ( async, I send the request and the receiver will get it even if it is now offline, as soon as it gets back online ) and enable message passing. Otherwise if using client server the slower server would slow down all the chain. The Queue does also load balancing so if a spike in load is perceived => we duplicate the slowest server and **load balance directly in the queue**,
