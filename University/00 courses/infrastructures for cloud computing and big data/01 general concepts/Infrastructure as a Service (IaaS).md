Infrastructure as a Service (IaaS) is a cloud computing model that provides **virtualized computing resources over the internet**, such as virtual machines, storage, and networking.

Instead of managing physical hardware, users rent infrastructure from a provider and manage the operating systems and applications themselves.

---

## Core idea

> “Rent infrastructure, manage everything above it”

The provider manages:
- Physical servers
- Storage hardware
- Networking hardware
- Virtualization layer

The user manages:
- Operating systems
- Applications
- Runtime environments
- Data

---

## Examples of IaaS providers

- OpenStack (private/public cloud)
- AWS EC2
- Google Compute Engine
- Microsoft Azure VMs

---

## Key characteristics

- Highly **scalable**
- **Pay-as-you-use model**
- **Hardware abstraction**
- Full control over OS
- Foundation for PaaS and SaaS layers
