In a worm approach there's **no original planning** on where to put the resources, the system is really dynamic.
There's a worm in the system that starts in a certain machine and dynamically **grows** (based on its implementation) exploring close machines.

To find other close nodes the worm uses **probe signals** to which only interested nodes will respond.

When a machine has been "infected by the worm" it communicates with the other worm segments in order to balance the load and coordinate the expansion.
